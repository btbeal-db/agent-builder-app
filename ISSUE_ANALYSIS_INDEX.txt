ISSUE: Field name editing feels unresponsive/sluggish

ROOT CAUSE: Fully controlled input that updates parent state on every keystroke

LOCATION: /Users/brennan.beal/code/agent-builder-app/frontend/src/components/StatePanel.tsx
          Lines 146-154 (name input field)

WHAT'S BROKEN:
  ✗ Input feels sluggish and unresponsive
  ✗ Each keystroke has visible lag (2-5ms)
  ✗ Input may lose focus between characters
  ✗ User experiences "fighting against the input"

WHAT'S NOT BROKEN:
  ✓ Data updates correctly in parent state
  ✓ Field names persist correctly
  ✓ Parent component receives updates
  ✓ API works fine
  ✓ State management is correct

WHY IT HAPPENS:
  Every keystroke triggers this cascade:
    onChange → updateField() → parent state update → parent re-render → 
    StatePanel re-render → input re-render → lag perceived by user

SOLUTION: Use local input state, update parent on blur

THE FIX:
  1. Add: editingNames state variable
  2. Add: getEditingName(), setEditingName(), saveEditingName() functions
  3. Update: toggleField() function
  4. Update: name input onChange/onBlur/onKeyDown handlers

EFFORT: ~15 minutes (experienced) to ~30 minutes (learning)
RISK: Very low (isolated component change)
IMPACT: High (significant UX improvement)
FILES: 1 file to modify (StatePanel.tsx)
